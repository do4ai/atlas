# 2. 구매자 storefront 화면 기획
앞 페이지에서 우리는 storefront의 IA와 라우트 맵을 정리했습니다. 이제 그 뼈대 위에 실제 구매자 화면을 올려야 합니다. 이번 단계의 목표는 단순히 화면 목록을 나열하는 것이 아닙니다. 사용자가 상품을 찾고, 이해하고, 구매를 결심하고, 주문을 완료하고, 결과를 확인하는 흐름이 끊기지 않도록 화면 책임을 정리하는 것입니다.

이커머스에서 storefront는 가장 중요한 사용자 경험 영역입니다. 사용자는 대부분 이 공간만 보고 서비스를 판단합니다. 그래서 화면 설계는 예쁜 카드와 배너를 만드는 문제가 아니라, 구매 결정에 필요한 정보를 적절한 시점에 적절한 밀도로 보여 주는 문제입니다.

이번 교재에서 storefront는 아래 한 줄의 흐름을 완성하는 데 초점을 둡니다.

> 상품 탐색 -> 상품 이해 -> 장바구니 구성 -> 주문/결제 -> 주문 확인

이 한 줄이 자연스럽게 이어지면 MVP 기준의 구매 경험은 성립합니다.

# 왜 storefront 화면 기획이 중요한가
프론트엔드 구현에서 흔히 생기는 문제는 화면 단위로만 생각하고 흐름 단위로 보지 않는 것입니다.

예를 들어:

- 상품 목록은 잘 만들었는데 상세로 넘어가면 정보 밀도가 갑자기 무너진다
- 장바구니는 있는데 로그인 전후 흐름이 어색하다
- 주문서는 있지만 사용자가 지금 무엇을 확인해야 하는지 우선순위가 없다
- 주문 완료 후 어디를 봐야 하는지 애매하다

이런 문제는 대부분 화면 자체의 문제가 아니라, 화면 간 역할 분리가 흐렸기 때문에 생깁니다. 그래서 이번 페이지에서는 먼저 각 화면이 "어떤 결정을 돕는 화면인가"를 정리합니다.

# 이 페이지에서 정할 것
이 페이지를 읽고 나면 아래 항목이 정리되어 있어야 합니다.

- storefront 핵심 화면 목록
- 화면별 주요 목적
- 비회원과 회원의 흐름 차이
- 빈 상태, 오류 상태, 예외 흐름
- 화면 간 전이 기준

이 문서가 정리되면 다음 단계의 프론트 구현 순서가 자연스럽게 나옵니다.

# 1. 구매자 storefront의 핵심 목표
구매자 화면은 아래 다섯 가지 질문에 답할 수 있어야 합니다.

- 무엇을 살 수 있는가
- 이 상품이 나에게 맞는가
- 얼마를 사게 되는가
- 지금 주문이 가능한가
- 주문 결과가 어떻게 되었는가

이 질문에 대응하는 화면이 바로 storefront의 핵심 화면입니다.

# 2. 이번 교재의 storefront 핵심 화면
이번 MVP에서 반드시 설계해야 하는 화면은 아래와 같습니다.

- 홈 또는 카탈로그 진입 화면
- 상품 목록 화면
- 상품 상세 화면
- 로그인/회원가입 화면
- 장바구니 화면
- 주문/결제 화면
- 주문 완료 또는 주문 상세 확인 화면

이번 교재에서는 `마이페이지 전체`, `찜 목록`, `리뷰 관리`, `쿠폰 센터` 같은 부가 화면은 넣지 않습니다. 핵심 구매 흐름 완성이 우선입니다.

# 3. 홈 또는 카탈로그 진입 화면
이 화면은 사용자가 처음 들어왔을 때 상품 탐색으로 자연스럽게 넘어가게 만드는 entry point입니다.

## 목적
- 서비스가 무엇을 파는지 빠르게 보여 준다
- 사용자가 상품 목록으로 진입하게 한다
- 주요 카테고리 또는 추천 진입점을 제공한다

## 화면에서 꼭 보여야 하는 것
- 상단 탐색 바
- 대표 상품 또는 카테고리 진입
- 검색 또는 탐색 시작점
- 로그인/장바구니 진입

## 과하게 넣지 말아야 하는 것
- 과도한 마케팅 섹션
- 긴 브랜드 스토리
- 구매 흐름과 무관한 복잡한 배너 회전

이번 교재는 실제 커머스 운영보다 학습용 MVP이므로, 홈은 화려함보다 진입 효율이 중요합니다.

# 4. 상품 목록 화면
상품 목록 화면은 사용자가 "무엇을 볼지"를 좁혀 나가는 공간입니다.

## 목적
- 상품 후보를 빠르게 비교한다
- 상세 페이지로 자연스럽게 이동한다
- 상품 탐색 실패를 줄인다

## 꼭 필요한 요소
- 상품 카드 리스트
- 가격
- 간단한 상품 설명 또는 태그
- 카테고리/필터 또는 정렬 중 최소 하나
- 상품 상세 진입 CTA

## 설계 원칙
- 정보는 비교 가능하게 반복되어야 한다
- 카드는 상세 진입을 돕는 수준까지 정보를 보여야 한다
- 카드에서 모든 정보를 다 해결하려 하지 않는다

## 빈 상태
- 검색/필터 결과 없음

### 보여 줄 것
- "조건에 맞는 상품이 없습니다" 안내
- 필터 초기화 또는 목록으로 돌아가기

## 오류 상태
- 목록 조회 실패

### 보여 줄 것
- 재시도 버튼
- 너무 기술적인 오류 문구는 피한다

# 5. 상품 상세 화면
상세 화면은 사용자가 실제 구매 결정을 하는 가장 중요한 화면입니다.

## 목적
- 상품을 이해하게 한다
- 구매 가능 여부를 판단하게 한다
- 장바구니 담기 행동으로 연결한다

## 꼭 필요한 요소
- 상품명
- 가격
- 설명
- 옵션 또는 기본 선택 상태
- 재고 또는 구매 가능 여부
- 장바구니 담기 버튼
- 챗봇 진입 또는 문의 CTA

## 상세 화면의 핵심 원칙
- 사용자가 "지금 이 상품을 살지 말지" 판단하는 데 필요한 정보가 우선이다
- 너무 많은 운영용 정보나 부가 정보는 뒤로 밀어야 한다

## 비회원과 회원의 차이
- 비회원도 상세는 볼 수 있어야 한다
- 장바구니 담기 또는 주문 전 단계에서 로그인 유도가 필요할 수 있다

## 오류 상태
- 상품이 존재하지 않음
- 조회 실패

### 보여 줄 것
- 목록으로 돌아가기
- 다시 시도하기

# 6. 로그인/회원가입 화면
이번 교재에서 인증 화면은 독립 목적이 아니라 구매 흐름을 이어 주는 관문입니다.

## 목적
- 장바구니/주문으로 들어가기 위한 인증 완료
- 사용자가 현재 하려던 행동으로 돌아가게 한다

## 핵심 원칙
- 로그인 이후 원래 보던 화면이나 다음 단계로 자연스럽게 복귀해야 한다
- 인증 화면이 흐름을 끊는 별도 목적지처럼 느껴지면 안 된다

## 꼭 필요한 요소
- 이메일/비밀번호 입력
- 회원가입 진입
- 로그인 성공 후 돌아갈 위치 유지

## 자주 놓치는 점
- `/cart` 또는 `/checkout` 진입 전 로그인 유도 후, 다시 원래 흐름으로 보내 주는 처리

# 7. 장바구니 화면
장바구니는 사용자의 임시 구매 의사를 정리하는 화면입니다.

## 목적
- 무엇을 담았는지 확인한다
- 수량을 수정한다
- 총액을 확인한다
- 주문 단계로 넘어간다

## 꼭 필요한 요소
- 담긴 상품 목록
- 상품별 수량
- 수량 변경/삭제
- 총액
- 주문하기 CTA

## 화면의 역할
장바구니는 단순 목록 화면이 아닙니다. "지금 이 조합으로 주문할 것인가"를 확인하는 화면입니다.

## 빈 상태
- 장바구니가 비어 있음

### 보여 줄 것
- 비어 있다는 명확한 메시지
- 상품 목록으로 돌아가는 CTA

## 오류 상태
- 수량 변경 실패
- 유효하지 않은 상품 상태

### 보여 줄 것
- 어떤 항목이 문제가 있는지 명확하게 알려 주기

# 8. 주문/결제 화면
이 화면은 구매 흐름의 마지막 핵심 단계입니다. 정보 입력보다 확인이 더 중요합니다.

## 목적
- 주문 내용을 확인한다
- 배송/결제 관련 필수 정보를 입력한다
- 최종 주문을 생성한다
- 결제 성공/실패 분기를 명확히 처리한다

## 꼭 필요한 요소
- 주문 상품 요약
- 수량과 총액
- 배송 정보 입력
- 결제 수단 또는 결제 진행 버튼
- 최종 제출 CTA

## 설계 원칙
- 사용자는 이 화면에서 "무엇을 사는지"와 "얼마를 결제하는지"를 다시 확인할 수 있어야 한다
- 입력 필드는 최소화하고, 확인 정보는 명확히 보여야 한다

## 성공 흐름
- 주문 생성
- 결제 성공
- 주문 완료/상세 화면으로 이동

## 실패 흐름
- 결제 실패
- 재시도 또는 장바구니/주문 단계로 복귀

이 실패 흐름을 화면에서 분명히 보여 주지 않으면 사용자는 "결제가 됐는지 안 됐는지"를 헷갈리게 됩니다.

# 9. 주문 완료 또는 주문 상세 확인 화면
주문 완료 화면은 단순 축하 메시지 화면이 아닙니다. 사용자가 "내 주문이 기록되었다"는 확신을 얻는 화면입니다.

## 목적
- 주문이 생성되었음을 보여 준다
- 주문 상태를 확인시킨다
- 다음 행동을 제시한다

## 꼭 필요한 요소
- 주문 번호 또는 주문 식별 정보
- 현재 주문 상태
- 주문 상품 요약
- 주문 목록 또는 홈으로 가는 CTA

## 설계 원칙
- 완료 메시지만 던지고 끝내면 안 된다
- 사용자가 다시 돌아와 확인할 수 있는 진입점을 같이 제공해야 한다

# 10. 비회원과 회원 흐름 차이
이번 MVP에서 가장 중요한 흐름 차이는 인증 시점입니다.

## 비회원이 가능한 것
- 홈 진입
- 상품 목록 조회
- 상품 상세 조회

## 비회원이 막히는 지점
- 장바구니 또는 주문 단계 진입 시점

## 회원이 가능한 것
- 장바구니 구성
- 주문/결제
- 주문 상태 확인

따라서 화면 설계에서는 비회원이 어디까지 갈 수 있고, 어디서 로그인으로 전환되는지를 명확히 보여 줘야 합니다.

# 11. 챗봇은 storefront 어디에 붙는가
이번 교재에서 챗봇은 storefront의 핵심 보조 흐름입니다.

## 우선 노출 위치
- 상품 상세 화면
- 주문/결제 화면
- 주문 상세 확인 화면

## 이유
- 상품 상세: 상품 관련 질문 발생
- 주문/결제: 배송/결제/정책 질문 발생
- 주문 확인: 상태 문의 발생

챗봇은 사용자가 막히는 지점에 나타나야지, 모든 화면에서 같은 강도로 튀어나와서는 안 됩니다.

# 12. 화면 간 대표 전이 흐름
이번 교재의 storefront 대표 흐름은 아래처럼 정리할 수 있습니다.

1. 홈 또는 상품 목록 진입
2. 상품 상세 이동
3. 장바구니 담기
4. 필요 시 로그인
5. 장바구니 확인
6. 주문/결제
7. 주문 완료 또는 주문 상세 확인

이 일곱 단계가 storefront MVP의 기본 골격입니다.

# 13. 실습용 storefront 화면 목록
이번 교재를 따라 하는 학습자는 최소한 아래 화면 목록을 먼저 적어 두는 것이 좋습니다.

| 화면 | 목적 |
| --- | --- |
| 홈 | 카탈로그 진입 |
| 상품 목록 | 탐색과 비교 |
| 상품 상세 | 구매 판단 |
| 로그인/회원가입 | 인증 |
| 장바구니 | 임시 구매 상태 확인 |
| 주문/결제 | 최종 확인과 제출 |
| 주문 완료/상세 | 결과 확인 |

# 14. 자주 하는 실수
- 상품 목록과 상세 화면의 역할을 구분하지 않는다
- 장바구니를 단순 중간 페이지처럼 만든다
- 주문/결제 화면에서 확인 정보보다 입력을 과하게 늘린다
- 주문 완료 후 사용자가 무엇을 해야 하는지 안내하지 않는다
- 비회원/회원 흐름 차이를 나중에 처리하려 한다

이 다섯 가지 실수는 storefront 구현 후반에 UX 부채로 남습니다.

# 정리
이 페이지에서 우리는 구매자 storefront의 핵심 화면과 각 화면의 책임을 정리했습니다. 이제 홈, 상품 목록, 상세, 장바구니, 주문/결제, 주문 확인까지의 흐름이 선명해졌습니다. 다음 페이지에서는 운영자 admin 화면을 따로 설계해, 구매자 경험과 운영 경험의 차이를 더 분명하게 드러냅니다.
