# 34강. self-supervised learning과 표현학습

이 강의의 목표는 라벨 없이 데이터 자체에서 학습 문제를 만들어 표현을 배우는 법을 아주 기초부터 이해하는 것입니다.

먼저 오늘의 핵심 식을 봅니다.

$$
\operatorname{sim}(z_i,z_i^+)>\operatorname{sim}(z_i,z_j)
$$

이 식을 외우기 전에, 식 안의 말과 기호를 먼저 하나씩 풀어야 합니다.

# 1. 먼저 오늘 쓸 말을 정리하자

| 말 | 뜻 |
|---|---|
| 표현 | 데이터를 모델이 쓰기 좋은 벡터로 바꾼 것 |
| 라벨 | 사람이 붙인 정답 이름 |
| self-supervised learning | 데이터 자체로 학습 문제를 만드는 방법 |
| positive pair | 같은 의미로 보아 가까워질 쌍 |
| negative pair | 다른 의미로 보아 멀어질 쌍 |

수학에서 어려운 부분은 계산보다 읽기입니다. 뜻을 모르고 계산하면 공식이 암호처럼 보입니다.

# 2. 먼저 떠올릴 장면

사진 하나를 자르고 색을 바꾸어도 같은 사진의 다른 모습입니다. 모델은 이런 두 표현을 가깝게, 다른 사진은 멀게 만들며 의미 있는 벡터를 배울 수 있습니다.

# 3. 핵심 식을 천천히 읽어 보자

- 왼쪽은 모델이 구하거나 비교하려는 값입니다.
- 오른쪽은 그 값을 만들기 위한 계산 구조입니다.
- 기호를 하나씩 말로 바꾸면 공식은 절차가 됩니다.
- 이 절차가 큰 모델 안에서 반복됩니다.

즉 핵심 식은 단순히 기호를 나열한 것이 아니라, 무엇을 보고 무엇을 계산할지 알려 주는 문장입니다.

# 4. 왜 이런 생각이 필요한가

AI 수학에서는 숫자가 하나만 나오지 않습니다. 벡터, 행렬, 확률분포, 손실함수처럼 많은 값이 한꺼번에 움직입니다.

그래서 self-supervised learning과 표현학습에서는 복잡한 대상을 바로 외우지 않고, 작은 단위로 쪼개어 읽습니다. 이 관점이 있어야 나중에 큰 모델의 식도 따라갈 수 있습니다.

# 5. 계산 순서로 다시 보기

1. 원본 데이터에서 변형을 만든다.
2. 같은 의미의 쌍을 positive로 둔다.
3. 다른 데이터는 negative로 둔다.
4. 표현 사이의 비슷도를 계산한다.
5. positive는 가깝게 negative는 멀게 학습한다.

이 순서를 말로 설명할 수 있으면, 공식을 완전히 외우지 않아도 다시 만들어 낼 수 있습니다.

문장에서 일부 단어를 가리고 맞히는 것도 라벨 없이 만든 학습 문제입니다.

$$
